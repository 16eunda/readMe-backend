🏷️ 3. analysisStatus 상태값 정리

상태의미누가 설정?다음 동작
PENDING 분석 대기 FileService (비프리미엄 저장 시) 프리미엄 구독 후 책 읽거나 ai-info 요청 시 큐에 등록
QUEUED 큐에 등록됨 FileService (프리미엄 저장/읽기/조회 시) AnalysisWorker가 5초 이내 처리
PROCESSING 분석 진행 중 AnalysisService (API 호출 직전) 완료 시 DONE, 실패 시 FAILED
DONE 분석 완료 AnalysisService (성공) / FileService (복사) 최종 상태, 더 이상 처리 없음
FAILED 분석 실패 AnalysisService (API 에러) 다음 번 책 읽기/조회 시 재시도 가능
LIMIT_EXCEEDED 일일 한도 초과 AnalysisService (10회 초과) 다음 날 자동 재시도 가능
PREMIUM_REQUIRED 프리미엄 필요 FileService (ai-info 응답용) DB에 저장 안 됨, 프론트 응답에만 사용

🎯 4. 추천 흐름 (RecService)

추천은 이미 분석 완료된 파일만 사용합니다. AI 분석을 직접 호출하지 않습니다.

👤 로그인 사용자 추천 (getRecommendationsForUser)
RecController.java → GET /recommendations → RecService.java
프론트에서 추천 요청 (GET /recommendations)
Header: Authorization + X-Device-Id
↓
1️⃣ 장르 기반 추천 (최대 3개)
최근 읽은 10개 파일에서 선호 장르 추출 → 같은 장르의 안 읽은 파일 추천
findTop10ByUserId...OrderByLastReadAtDesc → 장르 카운팅 → 최다 장르
findByAiGenreAndLastReadAtIsNullAndUserId(장르, userId)
⚠️ "미분류" 장르는 제외됨
↓ 아직 5개 미만이면
2️⃣ 별점 기반 추천 (빈자리 채움)
별점 4점 이상 파일들의 장르 수집 → 1단계와 다른 장르의 안 읽은 파일 추천
findHighRatedFilesByUserId(userId, 4) → 장르 수집
1단계 장르 제외 → 각 장르에서 1개씩
↓ 아직 5개 미만이면
3️⃣ 읽다 만 파일 추천
progress 10%~90% 사이인 파일 추천
findByProgressBetweenAndUserId(0.1, 0.9, userId)
↓ 아직 5개 미만이면
4️⃣ 폴백: 오래 전에 읽은 파일
"다시 읽어보세요!" — 가장 오래전에 읽은 파일 재추천
findOldestReadFilesByUserId(userId)
↓
중복 제거 후 최대 5개 반환
👻 게스트 사용자 추천 (getRecommendationsForDevice)
1️⃣ 장르 기반 추천 (최대 3개)
로그인과 동일한 로직 (deviceId 기반)
↓
2️⃣ 읽다 만 파일 추천
progress 10~90%
게스트는 별점 데이터 부족 → 2단계(별점 기반) 생략
↓
3️⃣ 폴백: 오래 전에 읽은 파일
재추천
↓
중복 제거 후 최대 5개 반환

🔒 5. AI 분석 일일 제한 (하루 10회)

AiAnalysisLog.java → 엔티티 (누가, 언제, 어떤 파일 분석했는지 기록)
AiAnalysisLogRepository.java → countTodayByUserId() / countTodayByDeviceId()
AnalysisService.java → canAnalyzeToday() / getRemainingToday()
항목설명
일일 한도10회 (DAILY_LIMIT 상수로 관리, 나중에 변경 가능)
카운트 대상Gemini API를 실제 호출한 경우만 (복사는 횟수에 미포함)
리셋 시점매일 자정 (00:00:00) 기준
한도 초과 시analysisStatus = LIMIT_EXCEEDED → 다음 날 자동 재시도 가능
남은 횟수 조회AnalysisService.getRemainingToday(user, deviceId)

💰 8. AI 비용 절약 전략 요약

현재 적용된 절약 방법들

  • 🔒 프리미엄만 분석 — 비프리미엄 유저는 AI 호출 0회
  • ♻️ 중복 제목 복사 — 같은 책이면 API 호출 없이 기존 결과 복사 (횟수 차감 없음)
  • 📊 일일 10회 제한 — 한 유저가 하루에 무한히 호출하는 것 방지
  • ⚙️ 비동기 큐 처리 — API 응답 대기 없이 바로 사용자에게 응답
  • 📖 3가지 시점 분석 — 파일 저장, 책 읽기, AI 정보 조회 시에만 트리거 (불필요한 호출 없음)
시나리오API 호출 횟수
비프리미엄 유저가 파일 100개 업로드0회
프리미엄 유저가 파일 10개 업로드 (모두 새 책)10회 (일일 한도)
프리미엄 유저가 파일 10개 업로드 (5개는 이미 분석된 제목)5회 (나머지 5개 복사)
이미 분석된 파일의 AI 정보 조회0회 (DB에서 반환)
추천 요청0회 (분석된 데이터만 사용)
